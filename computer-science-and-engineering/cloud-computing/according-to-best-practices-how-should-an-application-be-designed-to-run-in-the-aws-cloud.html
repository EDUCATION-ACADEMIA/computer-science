CLOUD COMPUTING

[SOURCES]
According to best practices, how should an application be designed to run in the AWS Cloud?

(A) Use tightly coupled components.

(B) ** Use loosely coupled components.

(C) Use infrequently coupled components.

(D) Use frequently coupled components.

EXPLANATIONS BELOW

Concept note-1: -According to best practices, how should an application be designed to run in the AWS Cloud? A. Use tightly coupled components.

Concept note-2: -Loosely coupled simulations can be deployed with end-to-end services and solutions such as AWS Batch and AWS ParallelCluster, or through a combination of AWS services, such as Amazon Simple Queue Service (Amazon SQS), Auto Scaling, AWS Lambda, and AWS Step Functions.

Concept note-3: -Amazon SNS is used for building and integrating loosely-coupled, distributed applications. SNS provides instantaneous, push-based delivery (no polling).
