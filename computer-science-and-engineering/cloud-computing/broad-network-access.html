Broad network access

(A) ** Cloud computing resources are accessible over the network

(B) Resource pooling

(C) Measured service

(D) Self-service

EXPLANATIONS BELOW

Concept note-1: -Broad network access means that the cloud customer or tenant, whoeverâ€™s consuming a service, does not manage the IT service infrastructure. And as a result, this means that that IT service runs in a provider data center somewhere and is therefore accessible to the consumer over a network.

Concept note-2: -According to NIST definition of cloud computing, it has five characteristics: on-demand self-service, broad network access, resource pooling, rapid elasticity, and measured service, while mobile computing focuses on device mobility and context awareness considering networking and mobile resource/data access.

Concept note-3: -There are also three main types of cloud computing services: Infrastructure-as-a-Service (IaaS), Platforms-as-a-Service (PaaS), and Software-as-a-Service (SaaS).

Concept note-4: -Static environment is not an essential characteristics of cloud computing.

Concept note-5: -2. Broad network access. Cloud computing resources are available over the network and can be accessed by diverse customer platforms.
