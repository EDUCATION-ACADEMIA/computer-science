Which of the following components reside on a NameNode?

(A) Filenames, blocks and checksums

(B) Blocks and heartbeat messages

(C) ** Filenames, block locations

(D) Blocks and block locations

EXPLANATIONS BELOW

Concept note-1: -HDFS Namenode stores meta-data i.e. number of data blocks, file name, path, Block IDs, Block location, no. of replicas, and also Slave related configuration. This meta-data is available in memory in the master for faster retrieval of data.

Concept note-2: -The hdfs-site. xml file contains information such as the value of replication data, namenode path, and datanode paths of your local file systems.

Concept note-3: -HDFS comprises of 3 important components-NameNode, DataNode and Secondary NameNode. HDFS operates on a Master-Slave architecture model where the NameNode acts as the master node for keeping a track of the storage cluster and the DataNode acts as a slave node summing up to the various systems within a Hadoop cluster.

Concept note-4: -Namenode does not contain actual data of files. Namenode stores metadata of actual data like Filename, path, number of data blocks, block IDs, block location, number of replicas and other slave related informations. Namenode manages all the request(read, write) of client for actual data file.

Concept note-5: -Namenode maps these blocks to DataNodes.
