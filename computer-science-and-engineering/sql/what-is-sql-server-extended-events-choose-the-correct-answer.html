What is SQL Server Extended Events. Choose the correct answer

(A) It is a performance handling system

(B) It is a bugs handling system

(C) ** It is an event-handling system

EXPLANATIONS BELOW

Concept note-1: -What is SQL Server extended events? SQL Server Extended Events is a performance monitoring tool that helps to collect and monitor the database engine actions to diagnose problems in SQL Server.

Concept note-2: -Extended Events is a lightweight performance monitoring system that enables users to collect data needed to monitor and troubleshoot problems in SQL Server.

Concept note-3: -XEL is the extension of the file name. Each time the event session is started, the system embeds a large integer into a new file name, to make the file name unique and different from the preceding occasion the session was started. The contents inside a . XEL are not plain text that can be viewed with Notepad.exe.

Concept note-4: -Benefits of SQL Server Extended Events Extended Events is a lightweight performance monitoring system that uses minimal performance resources. SQL Server Management Studio provides a graphical user interface for Extended Events to create and modify sessions and display and analyze session data.

Concept note-5: -This can be found by expanding the “Management” node with Object Explorer. You then expand “Extended Events” followed by “Sessions” which will give you a list of your sessions. After right-clicking on the session you want to view you can then select “Watch Live Data” as shown below.
