Which of the following is NOT a requirement of 1NF?

(A) It should only have single (atomic) valued attributes/columns.

(B) a Primary Key

(C) One item of data per column in one row (i.e. no repeating data)

(D) ** Data should be sorted in order

EXPLANATIONS BELOW

Concept note-1: -The order of Data does not matter. The order in which we store the data in a table does not matter in 1NF.

Concept note-2: -A table is in first normal form (1NF) if and only if all columns contain only atomic values-that is, each column can have only one value for each row in the table.

Concept note-3: -The first normal form states that: Every column in the table must be unique. Separate tables must be created for each set of related data. Each table must be identified with a unique column or concatenated columns called the primary key.

Concept note-4: -First normal form (1NF) is a property of a relation in a relational database. A relation is in first normal form if and only if no attribute domain has relations as elements. Or more informally, that no table column can have tables as values (or no repeating groups).
