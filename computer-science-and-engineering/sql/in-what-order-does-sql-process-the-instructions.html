In what order does SQL process the instructions?

(A) FROM-SELECT-WHERE-ORDER BY

(B) WHERE-FROM-SELECT-ORDER BY

(C) ** SELECT-FROM-WHERE-ORDER BY

(D) SELECT-FROM-ORDER By-WHERE

EXPLANATIONS BELOW

Concept note-1: -SQL queries adhere to a specific order when evaluating clauses, similar to how mathematical operations adhere to PEMDAS or BIDMAS. From the eyes of the user, queries begin from the first clause and end at the last clause. However, queries arenâ€™t actually read from top to bottom when carried out.

Concept note-2: -ORDER BY Characteristics: The ORDER BY clause is used to get the sorted records on one or more columns in ascending or descending order. The ORDER BY clause must come after the WHERE, GROUP BY, and HAVING clause if present in the query. Use ASC or DESC to specify the sorting order after the column name.

Concept note-3: -The ORDER BY keyword sorts the records in ascending order by default. To sort the records in descending order, use the DESC keyword.
