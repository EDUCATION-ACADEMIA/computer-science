To be in 2NF:

(A) ** Each non-key field relates to the primary key

(B) There are multiple entries in the same field

(C) There are no foreign keys

(D) All numbers are in decimal form

EXPLANATIONS BELOW

Concept note-1: -A table is in 2NF, only if a relation is in 1NF and meet all the rules, and every non-key attribute is fully dependent on primary key.

Concept note-2: -A relation is said to be in 2nd Normal Form in DBMS (or 2NF) when it is in the First Normal Form but has no non-prime attribute functionally dependent on any candidate keyâ€™s proper subset in a relation.

Concept note-3: -A table is in 2nd normal form (2NF) if it is in 1NF and all non-prime attributes are fully dependent on its primary key (no partial dependencies ). A table is in 3rd normal form (3NF) if it is in 2NF and it is free of transitive dependencies involving non-prime attributes .

Concept note-4: -A relational table is in third normal form (3NF) if it is already in 2NF and every non-key column is non transitively dependent upon its primary key. In other words, all nonkey attributes are functionally dependent only upon the primary key.
