What is default value of any column when it is missing or not known?

(A) ** NULL

(B) Empty

(C) 0

(D) Garbage Value

EXPLANATIONS BELOW

Concept note-1: -NULL values represent missing unknown data. By default, a table column can hold NULL values.

Concept note-2: -The default null value for null numeric fields is 0 (zero). The default null value for null alpha fields is either a single blank space ( ) or multiple blank spaces to length, depending on the fieldâ€™s requirements.

Concept note-3: -Default values can be NULL, or they can be a value that matches the data type of the column (number, text, date, for example).

Concept note-4: -If we do not provide any value for column allow NULL values, SQL Server assumes NULL as default value.

Concept note-5: -By default, a column can hold NULL values. The NOT NULL constraint enforces a column to NOT accept NULL values. This enforces a field to always contain a value, which means that you cannot insert a new record, or update a record without adding a value to this field.
