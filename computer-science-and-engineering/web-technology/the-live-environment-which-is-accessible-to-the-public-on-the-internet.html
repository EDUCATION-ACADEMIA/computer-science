EXPLANATIONS BELOW

Concept note-1: -Live Environment means the use of the Product in a customer environment under normal operating conditions, consistent with the Product specifications.

Concept note-2: -Simply put, a production environment is where the latest versions of software, products, or updates are pushed live to the intended users. Think of it as a final phase of production. This is the environment where the end user can see, experience, and interact with the new product.

Concept note-3: -This is the environment where the final deployment of the product is done. This is called a “Live” environment because customers interact with it live. Usually, access to the production server is limited and highly secure.

Concept note-4: -A production environment, sometimes called deployment environment, is the set of computers where finished, user-ready software is deployed and executed. When software code is moved to the production environment, it is the final step in a 4-tier architecture that includes development, testing, staging and production.
