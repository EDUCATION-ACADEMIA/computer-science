When a collision occurs with a hashing function, what is one method for dealing with the collisions?

(A) ** creating a new list at that index

(B) creating a new array

(C) creating a new hash table

(D) creating a new binary tree

EXPLANATIONS BELOW

Concept note-1: -If there is any collision (i.e. two different elements have same hash value) then store both the elements in the same linked list. The cost of a lookup is that of scanning the entries of the selected linked list for the required key.

Concept note-2: -One method for resolving collisions looks into the hash table and tries to find another open slot to hold the item that caused the collision. A simple way to do this is to start at the original hash value position and then move in a sequential manner through the slots until we encounter the first slot that is empty.
