When a pop() operation is called on an empty queue, what is the condition called?

(A) Overflow

(B) ** Underflow

(C) Syntax Error

(D) Garbage Value

EXPLANATIONS BELOW

Concept note-1: -When a pop() operation is called on an empty queue, what is the condition called? Answer-B) pop() on an empty queue causes Underflow. 17.

Concept note-2: -The process to add an element into queue is called Enqueue and the process of removal of an element from queue is called Dequeue.

Concept note-3: -Explanation: Underflow occurs when the user performs a pop operation on an empty stack.

Concept note-4: -Underflow Condition: When a stack is empty (i.e. TOP=-1) and we try to delete more element from it, then this condition is called underflow condition.

Concept note-5: -Simply put, overflow and underflow happen when we assign a value that is out of range of the declared data type of the variable. If the (absolute) value is too big, we call it overflow, if the value is too small, we call it underflow.
