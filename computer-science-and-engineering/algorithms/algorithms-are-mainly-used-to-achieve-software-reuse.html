Algorithms are mainly used to achieve Software Reuse

(A) ** YES

(B) NO

EXPLANATIONS BELOW

Concept note-1: -Types of software reuse include traditional “cut-and-paste” of software, application generators, COTS software, object-oriented programming, and software maintenance. Despite its benefits, careless application of software reuse can introduce faults.

Concept note-2: -In software development (and computer programming in general), code reuse, also called software reuse, is the use of existing software, or software knowledge, to build new software, following the reusability principles.

Concept note-3: -“Component-Based Software Engineering (CBSE) is a process that emphasizes the design and construction of computer-based systems using reusable software ‘components’ [7].” CBSE is a process that aims to design and construct software systems with reusable software components.

Concept note-4: -The term “reuse” implies the practice of using some piece of one process or product for making subsequent software products. Thus, reuse promotes creating software systems from existing software. As a result programmers need not always begin from scratch.
