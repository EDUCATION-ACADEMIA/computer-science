Many small sub-tasks together constitute a large ____

(A) sub-tasks

(B) sequence

(C) ** task

(D) algorithm

EXPLANATIONS BELOW

Concept note-1: -Subtasks are the additional steps that make up a task. They’re instrumental while working on massive projects with many different tasks. You must first choose the parent task before you can create a subtask, and you will be able to create as many subtasks as you need in the task view.

Concept note-2: -Subtasks are important because they help project managers accurately assign resources (such as project hours, personnel, and budget) to each task. They’re also great for forecasting since they give a clearer picture of where there might be conflicts within the project itself.

Concept note-3: -Subtasks are best used in instances where a project moves at different levels. For example, “Onboard New Client” is one large task, but there’s actually several moving pieces and teams just within that one task. It’s not uncommon for the original task at hand to be modified.

Concept note-4: -Meaning of subtask in English a task that forms part of a larger piece of work to be done: Each worker gets a subtask, processes it, and sends the result to their manager. When you have estimated how long each sub-task will take, you can identify when each of these tasks will be completed.
