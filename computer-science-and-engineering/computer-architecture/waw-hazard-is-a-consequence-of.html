COMPUTER SCIENCE AND ENGINEERING

COMPUTER ARCHITECTURE

[SOURCES]
WAW hazard is a consequence of:

(A) Data dependence

(B) Antidependence

(C) ** Output dependence

(D) Control dependence

EXPLANATIONS BELOW

Concept note-1: -WAW can be referred to as ‘Write after Write’. It is also known as Output Data dependency. If the later instruction tries to write on operand before earlier instruction writes it, in this case, the WAW hazards will occur. The condition to detect the WAW hazard is when On and On+1 both have a minimum one common operand.

Concept note-2: -Write-After-Write (WAW) Hazards A Write-After-Write hazard occurs when an instruction tries to write its result to the same register as a previously issued, but as yet uncompleted instruction.

Concept note-3: -There are three situations in which a data hazard can occur: read after write (RAW), a true dependency. write after read (WAR), an anti-dependency. write after write (WAW), an output dependency.

Concept note-4: -It’s “false” because the later instruction doesn’t actually need data from earlier instruction. The only conflict is finding a place to put the result. Like the slide says, “not a real data dependency"; only WAR hazards are true dependencies.

Concept note-5: -A dependence is a property of the instructions in a program, for example a true dependence arises when one instruction uses the value produced by an earlier instruction. A hazard is a potential problem in a pipeline that may arise from a dependence.
