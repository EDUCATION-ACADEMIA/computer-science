A high priority interrupt will wait for a low priority interrupt to finish.

(A) True

(B) ** False

EXPLANATIONS BELOW

Concept note-1: -High priority Interrupt: These interrupts cannot be interrupted. A high priority interrupt vector is located at 0008h in the program memory. Low priority Interrupt: These interrupts itself could be interrupted by high priority interrupts and its interrupt vector is located at 0018h.

Concept note-2: -Whenever an interrupt is generated by high priority process, the CPU stops executing the programâ€™s code and executes the code associated with the interrupt. When the CPU finishes executing the code associated with the interrupt, it resumes the execution of the code of the program that was running.

Concept note-3: -TRAP is the internal interrupt that has the highest priority among all interrupts except the divide by zero exception.

Concept note-4: -Hardware Interrupt: It happens when an external event occurs like an external interrupt pin changes its state from LOW to HIGH or HIGH to LOW. Software Interrupt: It happens according to the instruction from the software.
