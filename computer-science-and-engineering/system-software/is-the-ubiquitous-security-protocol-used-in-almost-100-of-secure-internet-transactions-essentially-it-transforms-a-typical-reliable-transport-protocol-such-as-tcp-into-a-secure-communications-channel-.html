Is the ubiquitous security protocol used in almost 100% of secure Internet transactions? Essentially, it transforms a typical reliable transport protocol (such as TCP) into a secure communications channel suitable for conducting sensitive transactions. (U25 KLO 3.4)

(A) Security Management

(B) ** Secured socket layer (SSL)

(C) The RSA encryption algorithm

(D) Policies

EXPLANATIONS BELOW

Concept note-1: -Hypertext transfer protocol secure (HTTPS) is the secure version of HTTP, which is the primary protocol used to send data between a web browser and a website. HTTPS is encrypted in order to increase security of data transfer.

Concept note-2: -SSL Stands for secure sockets layer. Protocol for web browsers and servers that allows for the authentication, encryption and decryption of data sent over the Internet.

Concept note-3: -The SSL protocol was originally developed by Netscape, to ensure security of data transported and routed through HTTP, LDAP or POP3 application layers.

Concept note-4: -Transport Layer Security (TLS) is a cryptographic protocol designed to provide communications security over a computer network. The protocol is widely used in applications such as email, instant messaging, and voice over IP, but its use in securing HTTPS remains the most publicly visible.
