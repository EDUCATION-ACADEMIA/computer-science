COMPUTER SCIENCE AND ENGINEERING

SOFTWARE ENGINEERING

[SOURCES]
A set of inputs, execution preconditions and expected outcomes is known as a

(A) Test plan

(B) ** Test case

(C) Test document

(D) Test Suite

EXPLANATIONS BELOW

Concept note-1: -“A Test Case is a set of inputs, execution preconditions, and expected outcomes developed for a particular objective, such as to exercise a particular program path or to verify compliance with a specific requirement.”

Concept note-2: -TCS – A document specifying inputs, predicted results, and a set of execution conditions for a test item.

Concept note-3: -“Regression test case” is not about the content of the test case, it is about the timing a test case is executed-if the functionality was tested once and was correct the next iteration is per definition a regression test.

Concept note-4: -1 Answer. For explanation: Boundary value analysis is a software testing technique in which tests are designed to include representatives of boundary values.

Concept note-5: -7. A testing strategy that test the application as a whole. Explanation: Validation testing tests the application as a whole against the user requirements. In system testing, it tests the application in the context of an entire system.
