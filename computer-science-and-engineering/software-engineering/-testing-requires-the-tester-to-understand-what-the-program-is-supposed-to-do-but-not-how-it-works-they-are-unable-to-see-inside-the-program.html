____ testing requires the tester to understand what the program is supposed to do, but not how it works, they are unable to “see inside” the program.

(A) validation

(B) ** Black box

(C) integration

(D) None

EXPLANATIONS BELOW

Concept note-1: -Black box testing (sometimes called “behavioral testing”) requires the tester to understand what the program is supposed to do, but not how it works – they are unable to “see inside” the program.

Concept note-2: -Black box testing involves testing a system with no prior knowledge of its internal workings. A tester provides an input, and observes the output generated by the system under test.

Concept note-3: -Black box testing-In this method of testing, the user/tester does not have any working knowledge of the internal structure of the software. It verifies the functionality of the software.

Concept note-4: -Black Box Testing mainly focuses on input and output of software applications and it is entirely based on software requirements and specifications. It is also known as Behavioral Testing.

Concept note-5: -The technique of testing without having any knowledge of the interior workings of the application is called black-box testing. The tester is oblivious to the system architecture and does not have access to the source code.
