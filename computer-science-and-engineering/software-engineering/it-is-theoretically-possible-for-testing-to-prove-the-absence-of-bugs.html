COMPUTER SCIENCE AND ENGINEERING

SOFTWARE ENGINEERING

[SOURCES]
*It is theoretically possible for testing to prove the absence of bugs.

(A) ** True

(B) False

EXPLANATIONS BELOW

Concept note-1: -In fact, finding a bug is often an indication that more serious bugs exist. So, while testing cannot prove the absence of bugs it can prove their existence-and this is an important and good thing. This allows the engineer to remove those detected bugs and, hence, increase confidence in the system.

Concept note-2: -Writing bug-free code is impossible. The best developers focus on making code that is reliable and maintainable, but anyone who says a piece of code is bug-free has not done their research. Research has shown that there is a factor of 20 between the productivity of the best and the worst programmers.

Concept note-3: -There can’t be an algorithm that proves the finding of all bugs, and you can’t quantify the number of absent bugs. This, then, implies that testing itself is undecidable.

Concept note-4: -This is because detecting all bugs in the software testing process is nearly impossible. Application bugs might appear due to the lack of time, resources, or simply improper input values and an arrow in validation scripts. A released software usually comes with minor bugs. Major and critical bugs come fixed.

Concept note-5: -Testing can only show the presence of errors, not there absence. Testing is intended to show that a program does what it is intended to do and to discover program defects before it is put into use. When you test software, you execute a program using artificial data.
