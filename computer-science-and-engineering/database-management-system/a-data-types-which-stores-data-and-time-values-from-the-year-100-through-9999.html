DATABASE MANAGEMENT SYSTEM

[SOURCES]
A data types which stores data and time values from the year 100 through 9999.

(A) Currency

(B) ** Date/Time

(C) OLE Objects

(D) Yes/No

EXPLANATIONS BELOW

Concept note-1: -The DATETIME data type stores an instant in time expressed as a calendar date and time of day. You select how precisely a DATETIME value is stored; its precision can range from a year to a fraction of a second.

Concept note-2: -The DATE data type stores the calendar date. DATE data types require four bytes. A calendar date is stored internally as an integer value equal to the number of days since December 31, 1899. Because DATE values are stored as integers, you can use them in arithmetic expressions.

Concept note-3: -TIMESTAMP Type The timestamp data type. The format is yyyy-MM-dd hh:mm:ss[. nnnnnnnnn]. Mapped to java.

Concept note-4: -The most widely used one is the DATETIME as it has been present since the earlier versions of SQL. SQL retrieves and displays DATETIME values in ‘YYYY-MM-DD hh: mm: ss’ format. The supported range is ‘1753-01-01 00:00:00’ to ‘9999-12-31 23:59:59.997’.
