COMPUTER SCIENCE AND ENGINEERING

COMPILER DESIGN

[SOURCES]
The character sequence forming a token is called as ____

(A) pattern

(B) symbol

(C) ** lexeme

(D) input

EXPLANATIONS BELOW

Concept note-1: -A lexeme is an actual character sequence forming a specific instance of a token, such as num. Pattern: A rule that describes the set of strings associated to a token. Expressed. as a regular expression and describing how a particular token can be formed.

Concept note-2: -Lexeme. A lexeme is a sequence of characters in the source program that matches the pattern for a token and is identified by the lexical analyzer as an instance of that token.

Concept note-3: -LEXEME-Sequence of characters matched by PATTERN forming the TOKEN. PATTERN-The set of rule that define a TOKEN. TOKEN-The meaningful collection of characters over the character set of the programming language ex:ID, Constant, Keywords, Operators, Punctuation, Literal String.

Concept note-4: -Detailed Solution. Lexeme: A lexeme is a sequence of characters in the source program that is matched by the pattern for a token.

Concept note-5: -Explanation: A lexeme is a string of characters that form a syntactic unit. It is reasonable to say that is the sequence of alphanumeric characters in a token.
