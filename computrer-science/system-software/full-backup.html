Full backup..

(A) Makes copies only of the new files

(B) Saves your backup to secondary storage

(C) ** Makes copies of all files

(D) Only allows you to make a double copy

EXPLANATIONS BELOW

Concept note-1: -A full backup is the process of creating one or more copies of all organizational data files in a single backup operation to protect them. Before the full backup process, a data protection specialist such as a backup administrator designates the files to be duplicated-or all files are copied.

Concept note-2: -Full backups The most basic and complete type of backup operation is a full backup. As the name implies, this type of backup makes a copy of all data to a storage device, such as a disk or tape.

Concept note-3: -Types of Backups Incremental backup: Backs up all files that have changed since the last backup occurred. Differential backup: Backs up only copies of all files that have changed since the last full backup.

Concept note-4: -Full Backup No matter what happens to your hardware, you know youâ€™ll have a complete copy of all the company information you need. Cons: Because these backups replicate so much information, they require a lot of storage space, time, and financial investment to complete.

Concept note-5: -Restoring a copy-only full backup is the same as restoring any other full backup. A copy-only log backup preserves the existing log archive point and, therefore, does not affect the sequencing of regular log backups. Copy-only log backups are typically unnecessary.
