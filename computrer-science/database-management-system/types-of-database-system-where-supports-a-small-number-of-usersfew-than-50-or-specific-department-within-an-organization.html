Types of Database System where supports a small number of users(few than 50) or specific department within an organization

(A) Enterprise Database

(B) Centralized Database

(C) ** Workgroup Database

EXPLANATIONS BELOW

Concept note-1: -multiuser DBMS-A database management system that supports multiple concurrent users. workgroup database-A multiuser database that supports a relatively small number of users (usually fewer than 50) or for a specific department in an organization.

Concept note-2: -Multi User Database: When the multiuser database supports a relatively small number of users (usually fewer than 50) or a specific department within an organization, it is called a workgroup database.

Concept note-3: -Workgroup database-Supports a small number of users (usually fewer than 50) or a specific department within an organization. Enterprise database-Used by an entire organization (usually 100â€™s of users) across many departments. Centralized database-Supports data located at a single site.

Concept note-4: -Classification Based on User Numbers A DBMS can be classification based on the number of users it supports. It can be a single-user database system, which supports one user at a time, or a multiuser database system, which supports multiple users concurrently.
