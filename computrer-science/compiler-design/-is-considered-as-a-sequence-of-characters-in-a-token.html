____ is considered as a sequence of characters in a token

(A) Pattern

(B) Token

(C) ** Lexeme

(D) DFA

EXPLANATIONS BELOW

Concept note-1: -A lexeme is a sequence of characters in the source program that is matched by the pattern for a token.

Concept note-2: -Explanation: The sequence of characters in the token is called a lexeme.

Concept note-3: -A string is a sequence of characters.

Concept note-4: -A token is an instance of a sequence of characters in some particular document that are grouped together as a useful semantic unit for processing.

Concept note-5: -In programming languages, a label is a sequence of characters that identifies a location within source code. In most languages, labels take the form of an identifier, often followed by a punctuation character (e.g., a colon).
