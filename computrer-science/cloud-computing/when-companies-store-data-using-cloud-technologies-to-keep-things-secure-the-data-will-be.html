CLOUD COMPUTING

[SOURCES]
When companies store data using cloud technologies, to keep things secure, the data will be

(A) Qualified

(B) Decrypted

(C) Protected

(D) ** Encrypted

EXPLANATIONS BELOW

Concept note-1: -Is your data encrypted when stored in the cloud? You will have to read the terms of service to find this out, but often your data will be stored on the cloud server with no encryption, this means that anyone that has (or can get) high level access to that server will be able to read your files.

Concept note-2: -Data stored in the cloud is typically encrypted, and anyone wanting to access that data needs to have the digital key. Also, big cloud computing companies have hired some of the world’s best data security experts.

Concept note-3: -There are two good ways to encrypt data that’s held on a cloud service: using a zero-knowledge service or encrypting your files before uploading them. Both methods work well, as long as the service doesn’t hold your encryption keys or password and uses AES 256-bit encryption or an equivalent to keep your privacy safe.

Concept note-4: -Which Cloud Storage Service Is End-to-End Encrypted? Sync.com, pCloud, Icedrive, MEGA and IDrive all offer end-to-end encryption, which protects your data from third-party intrusions when you transfer it from your device to cloud storage servers.
