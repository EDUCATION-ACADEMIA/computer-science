The types of AMI provided by AWS are

(A) Instance store backed

(B) EBS backed

(C) ** Both 1 & 2

(D) S3

EXPLANATIONS BELOW

Concept note-1: -Public, that grants instance launch permission to all AWS account holders. Explicit, that grants launch permission only to specific AWS accounts. Implicit, where only the AMI owner has permission to launch an instance.

Concept note-2: -An AMI stands for Amazon Machine Images. An AMI is a virtual image used to create a virtual machine within an EC2 instance. You can also create multiple instances using single AMI when you need instances with the same configuration.

Concept note-3: -AWS provides managed Amazon Machine Images (AMIs) that include various versions and configurations of Windows Server. In general, the AWS Windows AMIs are configured with the default settings used by the Microsoft installation media. However, there are customizations.

Concept note-4: -AWS provides you a combination of Infrastructure-as-a-Service (IaaS), Platform-as-a-Service (PaaS), and Software-as-a-Service (SaaS) offerings.
