A command that does not have an independent existence is called

(A) External command

(B) ** Internal command

(C) kernal

(D) shell

EXPLANATIONS BELOW

Concept note-1: -An internal command is the MS-DOS command stored in system memory and loaded from command.com or cmd.exe. Internal commands include MD, DATE, COPY, CD, TIME, COPR CON, TYPE, etc.

Concept note-2: -’Sys’ is not an internal command in DOS.

Concept note-3: -Internal commands are commands that are already loaded in the system. They can be executed any time and are independent. On the other hand, external commands are loaded when the user requests for them. Internal commands don’t require a separate process to execute them. External commands will have an individual process.

Concept note-4: -Internal Command: Internal commands are something which is built into the shell. For the shell built in commands, the execution speed is really high. It is because no process needs to be spawned for executing it. For example, when using the “cd” command, no process is created.
