Which of the following are the valid relationships in Use Case Diagrams

(A) Generalization

(B) Include

(C) Extend

(D) ** All of the mentioned

EXPLANATIONS BELOW

Concept note-1: -Explanation: Generalization, include, extend all of these are valid relationships in use case diagrams.

Concept note-2: -Explanation: Generalization, include, extend all of these are valid relationships in use case diagrams.

Concept note-3: -In UML modeling, a Include relationship is a relationship in which one use case (base use case) contains the functionality of another use case (inclusion use case). A containment relationship supports the reuse of functionality in the use case model.

Concept note-4: -The use case diagram displays three kinds of entities: the system, the actors, and the use cases. The system is represented by a box encompassing the use cases. The actors are users or systems that interact with the main system and are represented by figures.
