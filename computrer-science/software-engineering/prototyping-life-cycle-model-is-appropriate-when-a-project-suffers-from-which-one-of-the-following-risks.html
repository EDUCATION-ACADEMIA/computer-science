Prototyping life cycle model is appropriate when a project suffers from which one of the following risks?

(A) Schedule slippage

(B) Manpower turnover

(C) ** Incomplete and uncertain requirements

(D) Poor quality of outsourced work

EXPLANATIONS BELOW

Concept note-1: -Answer. Explanation: Advantages of Prototyping Model – This model is most appropriate for the projects that suffer from technical and requirements risks.

Concept note-2: -Spiral modelb. Prototyping modelc. RADd. Evolutionary modelCorrect Answer: dDetailed Solution: QUESTION 6:In the prototyping model of software development, which one of the following is the principalreason for constructing a prototype?

Concept note-3: -Waterfall. Through all development stages (analysis, design, coding, testing, deployment), the process moves in a cascade mode.

Concept note-4: -The prototype is a preliminary model that is refined until it meets end-user requirements. The process of repeating the steps to build a system over and over again is called an iterative process. Prototyping is more explicitly iterative than the conventional life cycle and it actively promotes system design changes.

Concept note-5: -Capability Maturity Model (CMM) specifies an increasing series of levels of a software development organization. The higher the level, the better the software development process, hence reaching each level is an expensive and time-consuming process. Unlike other models, it is not a life cycle model.
