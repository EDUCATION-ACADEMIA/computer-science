Which type of data hazard is not possible?

(A) WAR

(B) WAW

(C) ** RAR

(D) RAW

EXPLANATIONS BELOW

Concept note-1: -The correct answer is option 1 i.e RAR.

Concept note-2: -Read after read (RAR) is not a hazard case.

Concept note-3: -What types of data hazards would not appear in MIPS? Ans: Structural hazards: attempt to use the same resource in two different ways at the same time.

Concept note-4: -Data hazards occur if an instruction reads a Register that a previous instruction overwrites in a future cycle. We must eliminate data hazards or pipelining produces incorrect results. There are three ways to remove data hazards: The first way is to give the responsibility to software.

Concept note-5: -Types of Data Hazard in Pipelining The data hazard or dependency exists in the instructions in the three types that are as follows: Read after Write (RAW) Write after Read (WAR) Write after Write (WAW)
