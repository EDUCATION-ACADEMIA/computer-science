When the used CT routing, total communication time can be expressed by the following equation (1-number of communication lines used, m-the size of the message)

(A) tcomm = ts+(mtw+lh)l

(B) ** tcomm = ts + mtw + lh

(C) tcomm=(ts+mtw+lh)l

EXPLANATIONS BELOW

Concept note-1: -The time taken by the header of a message to travel between two directly-connected nodes in the network is called the per-hop time. It is also known as node latency.

Concept note-2: -Startup time (ts): Time spent at sending and receiving nodes (executing the routing algorithm, programming routers, etc.). Per-hop time (th): This time is a function of number of hops and includes factors such as switch latencies, network delays, etc. Also known as node latency.

Concept note-3: -Answer: The maximum number of tasks that can be executed simultaneously in a parallel program at any given time is known as its maximum degree of concurrency.

Concept note-4: -Message Passing Model. Message passing is the most commonly used parallel programming approach in distributed memory systems. Here, the programmer has to determine the parallelism. In this model, all the processors have their own local memory unit and they exchange data through a communication network.
