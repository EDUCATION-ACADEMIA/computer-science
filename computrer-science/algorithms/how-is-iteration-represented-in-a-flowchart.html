How is iteration represented in a flowchart?

(A) As a process

(B) As an input

(C) ** As a decision

EXPLANATIONS BELOW

Concept note-1: -With each iteration, a decision needs to be made as to whether to continue iterating or not. Decisions are represented as selection.

Concept note-2: -In flowcharts, a function call is shown as a box with double lines on its sides. The flowchart on the left calls the function on the right in one of the branches of its if .

Concept note-3: -Why is iteration important? Iteration allows algorithms to be simplified by stating that certain steps will repeat until told otherwise. This makes designing algorithms quicker and simpler because they don’t need to include lots of unnecessary steps.

Concept note-4: -Here, a hexagon shaped flowchart symbol is used to represent the for loop and the complete for loop statement is written inside this symbol.

Concept note-5: -Iteration is implemented in programming using FOR and WHILE statements. In programming, iteration is often referred to as ‘looping’, because when a program iterates it ‘loops’ to an earlier step. Each type of loop works in a slightly different way and produces different results.
